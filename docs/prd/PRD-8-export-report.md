# PRD-8: Downloadable Analysis Report

**Owner:** overnight agent
**Est:** 30 min
**Priority:** P1 — artifact judges can take with them
**Status:** pending

---

## Why

Judges see 20 demos in a row. They forget. A downloadable report is a physical artifact they keep. When they're deciding scores later, they have OUR report in front of them. Nobody else will have this.

## Design

"Export Report" button in the top stat cards area. On click, generates an HTML document and opens it in a new tab (or triggers download).

### Styling
- Inline CSS (no external dependencies)
- Light theme (white bg, dark text) — for printing/PDF
- Professional, minimal, clean typography
- Tables with borders
- Logo/branding in header and footer

### Implementation
1. Create `generateReport(pipeline, copilot)` function that returns HTML string
2. On button click: `window.open()` with the HTML as a data URL or blob URL
3. The report is purely client-side — no backend endpoint needed
